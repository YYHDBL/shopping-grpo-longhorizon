## 记录规则

1. 一条记录对应一个可以复现的实验问题，而不是一次随手运行。
2. 开头必须写清 code commit、模型/数据版本、环境和输出目录；密钥绝不写入文档。
3. 结果必须附固定 benchmark 清单和完整命令。运行失败也要记录失败位置与可复现证据。
4. “观察”与“结论”分开写：前者是日志/指标，后者是据此做出的下一步决定。
5. 还没有运行的段落填 `待运行`，不使用预计数值或“应该会提升”代替结果。
